# GDevelop 自定义 AI 快速配置指南

## 🚀 快速开始（推荐方法）

### 1. 启动 GDevelop

```bash
cd /workspace/GDevelop/newIDE/app
npm start
```

### 2. 打开浏览器访问

访问：http://localhost:3000

### 3. 配置自定义 AI

在浏览器中按 `F12` 打开控制台，输入：

```javascript
// 启用自定义 API（替换为你的 API 地址和密钥）
window.enableCustomAI('https://your-api.com/v1', 'your-api-key-here')
```

### 4. 刷新页面

配置完成后刷新页面，AI 功能就会使用你的自定义 API。

---

## 🔧 配置示例

### 配置 CNB OpenAI 代理

```javascript
window.enableCustomAI('https://your-cnb-proxy-url.com/v1', 'your-cnb-token')
```

### 配置本地 OpenAI 代理

```javascript
window.enableCustomAI('https://knc2ugzr28-8080.cnb.run/v1', 'your-local-key')
```

### 配置其他 AI 服务

```javascript
// DeepSeek
window.enableCustomAI('https://api.deepseek.com/v1', 'sk-deepseek-key')

// Anthropic（需要兼容层）
window.enableCustomAI('https://api.anthropic.com/v1', 'sk-anthropic-key')

// 其他 OpenAI 兼容服务
window.enableCustomAI('https://your-api.com/v1', 'your-api-key')
```

## 📝 查看配置

在控制台输入：

```javascript
// 查看当前配置
window.GDevelopCustomAIConfig
```

输出示例：
```javascript
{
  enabled: true,
  baseUrl: "https://your-api.com/v1",
  apiKey: "your-api-key"
}
```

---

## ❌ 禁用自定义 API

```javascript
window.disableCustomAI()
```

这会恢复使用官方 GDevelop AI 服务。

## 🔍 故障排除

### 问题：仍然要求登录

**解决方案：**
1. 检查配置是否正确加载：`window.GDevelopCustomAIConfig`
2. 确认 `enabled` 为 `true`
3. 刷新页面

### 问题：API 请求失败

**解决方案：**
1. 打开浏览器开发者工具（F12）→ Network 标签
2. 发送 AI 请求
3. 查看失败的请求详情
4. 检查：
   - API 地址是否正确
   - API Key 是否有效
   - CORS 是否配置正确
   - API 服务器是否运行

### 问题：配置不保存

**解决方案：**
1. 清除浏览器缓存
2. 重新运行配置命令
3. 检查浏览器是否允许 localStorage

## 🎯 核心功能

### 跳过登录检查

当 `customAiApiConfig.enabled` 为 `true` 时：

```javascript
const customConfig = getCustomAiApiConfig();
if (!customConfig.enabled && !profile) {
  throw new Error('User should be authenticated.');
}
```

### 动态 API 客户端

根据配置返回不同的 axios 实例：

```javascript
const getApiClient = () => {
  const customConfig = getCustomAiApiConfig();
  if (customConfig.enabled && customConfig.baseUrl) {
    return axios.create({
      baseURL: customConfig.baseUrl,
      headers: {
        'Authorization': `Bearer ${customConfig.apiKey}`,
        'Content-Type': 'application/json',
      },
    });
  }
  return axios.create({ baseURL: GDevelopGenerationApi.baseUrl });
};
```

### 跳过官方认证

当使用自定义 API 时，不添加官方认证头：

```javascript
const customConfig = getCustomAiApiConfig();
const authorizationHeader = customConfig.enabled ? '' : await getAuthorizationHeader();
```

## 🌟 使用自定义 API 的优势

2. ✅ 可以使用任何 OpenAI 兼容的 API
3. ✅ 完全控制 API 调用

---

## 📌 注意事项

1. **API 兼容性**：你的 API 必须完全兼容 OpenAI 接口
2. **端点要求**：需要支持 GDevelop 的 AI 端点
3. **安全性**：API Key 存储在 localStorage 中
4. **CORS**：确保你的 API 支持 CORS
